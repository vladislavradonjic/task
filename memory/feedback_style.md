Prefer full names over abbreviations in code. For example: `description` not `desc`, `properties` not `props`, `tokens` not `toks`.

**Why:** Author's stated preference — explicit is better than terse.

**How to apply:** Use full names when proposing or writing code. Exception: widely-used conventions where the short form is unambiguous and standard (e.g. `m` for a regex match object, `i` for a loop index, `f` for a file handle in a `with` block).
